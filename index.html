k近邻算法

发表于 2017-11-21 | 分类于 机器学习算法

1、主要思想：

有一个给定的训练集（即含有标签），对于一个输入的待分类的数据，计算该数据与训练集中每个样本的距离，选择距离最近的前k个样本，这k个样本中哪个类别最多，待分类数据就属于哪个类别。

2、特点：

1）无参数

2）监督学习

3、需要考虑：

1）k如何选择：

通过多次尝试，否则会发生过拟合或欠拟合的问题。

2）距离如何算：

欧式距离、曼哈顿距离、切比雪夫距离、马氏距离等

3）除了少数服从多数，还有其它方法吗?

4）对于给定的标签数据训练集、测试集的比例

5）查找前k个的算法：

除了简单的线性查找，还有其它算法，如kd树，BBF算法，​

4、缺点：

1）如果训练集很大，占据空间

2）如果特征维数多，占据时间​，产生维数灾难

sklearn--网格搜索与交叉验证

发表于 2017-11-21 | 分类于 sklearn

0、参考：

http://blog.csdn.net/JasonDing1354/article/details/50562522

http://www.cnblogs.com/nolonely/p/7007961.html

在参数中有一类是超参数，即不是通过训练得到的，而是由人工指定的，但是取哪个值需要不断的去尝试。

1、交叉验证

可以用来选择合适的超参数，对于每一个参数，进行n折交叉验证，得到n个结果然后进行平均，得到此超参数对应的结果。

2、网格搜索交叉验证

穷举所有可取的超参数，计算相应的结果。

  • sklearn.model_selection.GridSearchCV(model,param,cv,scoring)

    • model是定义的模型，如model=sklearn.svm.SVC()
    • param是要穷举的超参数，如param={‘C’:[0.1,1,10],’kernel’:[‘rbf’,’poly’,’linear’]}
    • cv是交叉验证的次数
    • scoring是评估指标，如accuracy，也可以是自己定义的评价指标

  • 有了train_x,train_y后

    grid=GridSearchCV()

    grid.fit(x,y)

  • grid中包括最后的结果

    • grid.grid_scores_

    • grid.best_scores_ grid.best_params_ grid.best_estimator

    • print grid.grid_scores_[0].parameters

      print grid.grid_scores_[0].cv_validation_scores

      print grid.grid_scores_[0].mean_validation_score

3、随机采样交叉验证

RandomSearchCV(estimator, param_distributions, n_iter=10, scoring=None, fit_params=None, n_jobs=1, iid=True, refit=True, cv=None, verbose=0, pre_dispatch='2*n_jobs', random_state=None, error_score='raise', return_train_score=True)

思想是不穷举所有的超参数，而是以一定的概率分布选择，速度更快

1）estimator还是模型

2）param_distributions是超参数

3）n_iter指明多少个参数组合被计算

4）cv指定交叉验证的次数

sklearn--logisticRegression

发表于 2017-11-21 | 分类于 sklearn

0、参考：

http://blog.csdn.net/cherdw/article/details/54891073

http://blog.csdn.net/sun_shengyun/article/details/53811483

1、官方api

http://scikit-learn.org/stable/modules/generated/sklearn.linear_model.LogisticRegression.html#sklearn.linear_model.LogisticRegression

2、参数解析

  • penalty：正则化参数，L1或者L2，L2是参数平方和，L1是参数和。为了解决过拟合，一般使用L2，若还过拟合，则使用L1（应该是L1的纠正程度更厉害一些）

  • C：正则化系数λ的倒数，默认是1

  • class_weight：dict or ‘balanced’, default: None。若为None，所有类的权重相同，若为balanced，则由系统自动计算每个类的权重，dict即自己指定每个类的权重，是一个字典的结构。

    在分类模型中，我们经常会遇到两类问题：

    第一种是误分类的代价很高。比如对合法用户和非法用户进行分类，将非法用户分类为合法用户的代价很高，我们宁愿将合法用户分类为非法用户，这时可以人工再甄别，但是却不愿将非法用户分类为合法用户。这时，我们可以适当提高非法用户的权重。

    第二种是样本是高度失衡的，比如我们有合法用户和非法用户的二元样本数据10000条，里面合法用户有9995条，非法用户只有5条，如果我们不考虑权重，则我们可以将所有的测试集都预测为合法用户，这样预测准确率理论上有99.95%，但是却没有任何意义。这时，我们可以选择balanced，让类库自动提高非法用户样本的权重。

  • solver：损失函数的优化方法{‘newton-cg’, ‘lbfgs’, ‘liblinear’, ‘sag’, ‘saga’},

    1）newton-cg:牛顿方法，利用损失函数二阶导数矩阵即海森矩阵来迭代优化损失函数。

    2）lbfgs：牛顿方法，利用损失函数二阶导数矩阵即海森矩阵来迭代优化损失函数。

    3）liblinear:使用了开源的liblinear库实现，内部使用了坐标轴下降法来迭代优化损失函数。

    4）sag：梯度下降法

    L1正则化是非连续的，而1、2、4都需要求导，所以只适用于L2，而liblinear适合于两者；
    同时小数据时使用liblinear较好，大数据时使用sag较好。
    多元逻辑回归包括一对多，多对多，liblinear只适用于一对多，但是多对多更精确一些。

  • multi_class：多分类，可以使用一对多（OvR）和多对多（MvM）

3、sklearn中进行交叉验证时，可以使用网格化来测试这些超参数，选择合适的超参数。

非原创--dos切换代码页

发表于 2017-11-20 | 分类于 windows系统

参考:http://blog.csdn.net/samlei/article/details/5494624

在中文Windows系统中，如果一个文本文件是UTF-8编码的，那么在CMD.exe命令行窗口（所谓的DOS窗口）中不能正确显示文件中的内容。在默认情况下，命令行窗口中使用的代码页是中文或者美国的，即编码是中文字符集或者西文字符集。

如果想正确显示UTF-8字符，可以按照以下步骤操作：

  1. 打开CMD.exe命令行窗口

  2. 通过 chcp命令改变代码页，UTF-8的代码页为65001,命令为chcp 65001，执行该操作后，代码页就被变成UTF-8了。但是，在窗口中仍旧不能正确显示UTF-8字符。

  3. 修改窗口属性，改变字体：在命令行标题栏上点击右键，选择”属性”->”字体”，将字体修改为True Type字体”Lucida Console”，然后点击确定将属性应用到当前窗口。
    这时使用type命令就可以显示UTF-8文本文件的内容了： type filename.txt

  4. 通过以上操作并不能完全解决问题，因为显示出来的内容有可能不完全。可以先最小化，然后最大化命令行窗口，文件的内容就完整的显示出来了。

其他代码页参考：

  • chcp 936 切换回简体中文，GBK编码

  • chcp 437 切换回英语，ISO8859编码

摆弄路由器之感

发表于 2017-11-20 | 分类于 windows系统

  1. 必要时就得使用reset键，坏不了。

  2. 路由器买来之后插上网线就可以使用了，不需要密码就能上网，处于AP模式，需要自己设置为路由器模式。

  3. 对于台式电脑，用2根网线进行设置：一根连计算机和路由器的lan口，一根连路由器的wan口和宽带上网口。对于笔记本，用1根：连路由器 的wan口和宽带上网口，然后
    笔记本搜到路由器的名字（默认为mac地址名）进行无线连接。然后进行设置，主要是进行加密，wifi的名字和密码设置。

  4. 设置多号同时登录。将lan中的启用DHCP服务器取消，重启路由器，然后网线不再连接wan口，连接lan口，即可
