非原创--dos切换代码页

发表于 2017-11-20 | 分类于 windows系统

参考:http://blog.csdn.net/samlei/article/details/5494624

在中文Windows系统中，如果一个文本文件是UTF-8编码的，那么在CMD.exe命令行窗口（所谓的DOS窗口）中不能正确显示文件中的内容。在默认情况下，命令行窗口中使用的代码页是中文或者美国的，即编码是中文字符集或者西文字符集。

如果想正确显示UTF-8字符，可以按照以下步骤操作：

  1. 打开CMD.exe命令行窗口

  2. 通过 chcp命令改变代码页，UTF-8的代码页为65001,命令为chcp 65001，执行该操作后，代码页就被变成UTF-8了。但是，在窗口中仍旧不能正确显示UTF-8字符。

  3. 修改窗口属性，改变字体：在命令行标题栏上点击右键，选择”属性”->”字体”，将字体修改为True Type字体”Lucida Console”，然后点击确定将属性应用到当前窗口。
    这时使用type命令就可以显示UTF-8文本文件的内容了： type filename.txt

  4. 通过以上操作并不能完全解决问题，因为显示出来的内容有可能不完全。可以先最小化，然后最大化命令行窗口，文件的内容就完整的显示出来了。

其他代码页参考：

  • chcp 936 切换回简体中文，GBK编码

  • chcp 437 切换回英语，ISO8859编码

摆弄路由器之感

发表于 2017-11-20 | 分类于 windows系统

  1. 必要时就得使用reset键，坏不了。

  2. 路由器买来之后插上网线就可以使用了，不需要密码就能上网，处于AP模式，需要自己设置为路由器模式。

  3. 对于台式电脑，用2根网线进行设置：一根连计算机和路由器的lan口，一根连路由器的wan口和宽带上网口。对于笔记本，用1根：连路由器 的wan口和宽带上网口，然后
    笔记本搜到路由器的名字（默认为mac地址名）进行无线连接。然后进行设置，主要是进行加密，wifi的名字和密码设置。

  4. 设置多号同时登录。将lan中的启用DHCP服务器取消，重启路由器，然后网线不再连接wan口，连接lan口，即可

github+hexo博客搭建

发表于 2017-11-16 | 分类于 搭建博客

一、所需工具
1、下载nodejs
2、下载git
二、安装
1、在github上创建repository：命名为username.github.io
2、安装hexo
打开git命令端，进入某个目录
1)npm install -g hexo-cli(这里可能出问题，墙的原因，可以试试这个http://npm.taobao.org/，即使用cnpm)
2)hexo init blog
3)cd blog
4)(c)npm install
5)hexo g
6)hexo s(可以在本地打开localhost:4000查看网站)
3、hexo和github绑定
1)安装插件(c)npm install hexo-deployer-git –save
2)blog目录下有_config文件，打开进行修改：
deploy:
type: git
repo: git@github.com:yourusername/yourusername.github.io.git
branch: master
3)hexo d
致此，完成博客的搭建，可以通过username.github.io进行访问，当然也可以自己买域名修改域名
三、进一步完善
1、修改主题
1)https://hexo.io/themes/这个网站有很多主题可供使用
2)选择喜欢的主题进入github页面进行clone到本地 blog/themes文件夹下
3)修个blog/_config配置文件，theme:(名称为下载的主题名)，可能有和主题相关的插件也需要下载
4)hexo s –debug可以在本地进行预览，若可以则hexo g，hexo d到远程
2、配置主题
有2个配置文件，blog下有一个配置文件，对应的主题下有一个配置文件
1)blog下的配置文件可以设置blog的名称，描述等等
2)对应主题下的配置文件可以设置关于主题的相关内容。
四、hexo相关命令
1、hexo new ‘a article’
新建一篇文档，然后去blog/source/posts下进行编辑
2、hexo s –debug
可以在本地进行debug，每次修改之后在localhost:4000/会看到更新
3、hexo g、hexo d
若更新满意，则要发布到远程仓库中，就使用这2个命令，然后username.github.io对应的网站就会进行更新。
